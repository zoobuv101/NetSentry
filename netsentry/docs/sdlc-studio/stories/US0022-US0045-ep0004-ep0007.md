**As a** network operator who has enabled AI identification **I want** NetSentry to submit low-confidence devices to Claude and use the result to identify them **So that** unusual devices that heuristics can't classify are still identified

- **When** response is parsed
- **Then** `IdentificationResult` row with `source="ai"`, `confidence=0.82`, `reasoning=<json>`

### AC3: Rate limit enforced (max 10 calls per cycle)
- **Given** 15 devices below confidence threshold in one scan cycle
- **When** pipeline runs
- **Then** only first 10 are submitted to Claude; remaining 5 retain heuristic result with note in reasoning

### AC4: Signal fingerprint cache prevents duplicate calls
- **Given** device with identical signals as a previous AI call (same port set, OUI, mDNS)
- **When** pipeline runs
- **Then** cached result used; no new API call made

### AC5: No PII in AI payload
- **Given** device with `friendly_name="Ian's iPhone"` and `current_ip="192.168.1.25"`
- **When** AI payload is constructed
- **Then** payload does NOT contain `friendly_name`, `current_ip`, `owner`, or any user-assigned strings; only OUI vendor, ports, mDNS type, OS family, hostname pattern, DNS domain patterns

### AC6: AI API error falls back to heuristic result
- **Given** Anthropic API returns 500 error
- **When** AI call is attempted
- **Then** heuristic result retained; WARNING logged; no crash

## Scope
- `netsentry/identification/ai.py` â€” `AiEnricher` with rate limiter, cache, payload builder

## Edge Cases & Error Handling

|----------|-------------------|

| API key invalid | `AnthropicAuthError`; integration marked unhealthy; feature disabled until restart |
| Claude returns invalid JSON | JSON parse error; heuristic result used |
| Rate limit window resets mid-scan | Counter resets at start of each scan cycle |
